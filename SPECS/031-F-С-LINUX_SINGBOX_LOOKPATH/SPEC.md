# Спецификация: Linux — поиск sing-box через PATH, затем каталог `bin/`

## Контекст

Пользователи ставят **sing-box из пакета дистрибутива** (например Fedora RPM → `/usr/bin/sing-box`). Лаунcher сейчас ожидает бинарник только в `<каталог_лаунчера>/bin/sing-box`, из‑за чего на вкладке Core показывается ошибка «sing-box not found», хотя ядро в системе есть (см. [issue #48](https://github.com/Leadaxe/singbox-launcher/issues/48)).

## Цель

На **Linux** определять исполняемый файл sing-box в таком порядке:

1. **`exec.LookPath("sing-box")`** — если в `PATH` найден исполняемый файл с таким именем, использовать **его полный путь** для запуска, проверки версии, подсказок по capabilities и отображения пути к ядру.
2. **Иначе** — текущее поведение: **`filepath.Join(ExecDir, "bin", sing-box)`** (локальный `bin/` рядом с бинарником лаунчера).

Платформы **Windows и macOS** — без изменений логики (как сейчас).

## Требования

- Порядок приоритета строго: сначала `LookPath`, при отсутствии — локальный `bin/`.
- Скачивание/установка ядра из лаунчера (**Core → download**) по‑прежнему кладёт бинарник в **локальный** `<ExecDir>/bin/`, а не перезаписывает системный путь из `LookPath`. Для этого в реализации нужно явно различать **путь запуска** (resolved) и **путь установки из приложения** (bundled `bin/`), если одного поля недостаточно.
- Логирование при выборе пути — уровень **Debug** через `internal/debuglog` (какой путь выбран и почему), без шума в обычном режиме.
- Язык UI и кода — **SPECS/CONSTITUTION.md**.

## Критерии приёмки

- [x] На Linux при установленном в `PATH` `sing-box` вкладка Core показывает версию и позволяет запуск без обязательной копии в `bin/` лаунчера.
- [x] На Linux без `sing-box` в `PATH` поведение как раньше: ожидание файла в `<ExecDir>/bin/sing-box`.
- [x] Установка ядра средствами лаунчера по-прежнему нацелена на локальный `bin/`, системный бинарник из пакета не затирается неожиданно.
- [x] `go build ./...`, `go test ./...`, `go vet ./...` проходят.

## Вне скоупа

- Переменные окружения и отдельное поле в UI «путь к sing-box» (возможный follow-up).
- Изменение приоритета «локальный `bin/` важнее системного» (в этой задаче приоритет за `LookPath`).
