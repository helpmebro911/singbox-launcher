# SPEC: Xray JSON-подписка — jump (`dialerProxy`) любого поддерживаемого типа

## Статус

**Завершено** (**036-F-C**). Follow-up к **`033-F-N-SUBSCRIPTION_XRAY_JSON_ARRAY`**.

Реализовано: hop **`socks`** и **`vless`**; остальные `protocol` у jump outbound по-прежнему дают пропуск элемента с **`WarnLog`**.

## 1. Проблема (исходная)

В полном конфиге Xray поле **`dialerProxy`** / **`dialer`** в `streamSettings.sockopt` — ссылка по **тегу** на **любой** другой outbound в том же элементе массива. На практике чаще всего это SOCKS, но встречаются сценарии, где промежуточный hop — **VLESS** и др.

До **036** поддерживался только **`protocol: socks`** у целевого outbound; иначе элемент массива пропускался.

## 2. Цель (достигнутая часть)

Расширить **`ParsedNode.Jump`** так, чтобы hop мог быть не только SOCKS: реализованы **`socks`** и **`vless`**; остальные типы — отдельные задачи при появлении маппинга Xray JSON → sing-box.

## 3. Требования

1. Если outbound по тегу `dialerProxy` — **SOCKS**, поведение остаётся как в **033** (обратная совместимость).
2. Если outbound — **другой поддерживаемый** тип: построить из него sing-box-shaped данные (аналог `ParsedNode` для hop или общая структура «hop»), сгенерировать JSON outbound(ы) **перед** основным VLESS, выставить цепочку **`detour`** в порядке, который ожидает sing-box (см. [Dial Fields — detour](https://sing-box.sagernet.org/configuration/shared/dial/)).
3. Теги hop’ов: стабильная схема уникальности (как сейчас `*_jump_server` / префиксы / `MakeTagUnique`), без коллизий между элементами массива и с основным тегом.
4. Неподдерживаемый протокол или битая ссылка — по-прежнему пропуск элемента + **`WarnLog`** / **`debuglog`** с контекстом (индекс элемента, тег).
5. **Share URI**, UI «Копировать ссылку» — явно зафиксировать: скорее всего «не поддерживается» для любой цепочки с `detour`, если уже так для SOCKS-jump (**033**).

## 4. Вне scope (пока)

- **Несколько** hop’ов подряд (цепочка `dialerProxy` → `dialerProxy`) — может потребовать `[]Jump` или рекурсивного обхода; описать отдельно в **PLAN** как второй этап или отдельную задачу.
- Полный паритет со всеми протоколами Xray и кастомным `routing` внутри элемента.

## 5. Критерии приёмки

1. Эталонный кейс: элемент массива с `dialerProxy` на **не-SOCKS** supported outbound → одна логическая нода в лаунчере, в `config.json` — корректная цепочка `detour`, sing-box принимает конфиг.
2. Регрессия: существующие тесты **033** (SOCKS jump) проходят.
3. Документация: **`docs/ParserConfig.md`**, при необходимости **`docs/ARCHITECTURE.md`**, **`docs/release_notes/upcoming.md`**.
