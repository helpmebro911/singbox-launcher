# Спецификация: после Update перечитывать конфиг через Clash API

Мелкая правка: после успешного нажатия кнопки **Update** (updateConfigButton) перечитывать настройки Clash API из обновлённого `config.json`, чтобы лаунчер подхватывал новый конфиг без рестарта sing-box.

## Проблема

Сейчас по нажатию **Update** выполняется только `RunParserProcess()`: подтягиваются подписки, пересобираются outbounds, записывается `config.json`. Состояние лаунчера (вкладка Servers, Clash API — baseURL, token, группы селекторов) при этом не обновляется из нового файла. Чтобы вкладка Servers и селекторы соответствовали обновлённому конфигу, пользователю приходится перезапускать sing-box или приложение.

## Цель

После успешного завершения Update перечитывать конфигурацию Clash API из обновлённого `config.json` (через существующий метод `APIService.ReloadClashAPIConfig()`), чтобы:

- лаунчер подхватывал актуальные `experimental.clash_api` (URL, secret) и список селекторов;
- не требовался рестарт sing-box или приложения для отображения актуального состояния.

## Требования

1. **После успешного RunParserProcess** — вызывать `ReloadClashAPIConfig()` (на контроллере: `ac.APIService.ReloadClashAPIConfig()`), чтобы перечитать из `config.json` настройки Clash API и при необходимости обновить `SelectedClashGroup` по умолчанию.
2. Вызов выполнять только при успешном завершении обновления конфига (в существующем пути успеха RunParserProcess / UpdateConfigFromSubscriptions).
3. Ошибку `ReloadClashAPIConfig` не показывать пользователю как блокирующую (конфиг уже сохранён); при необходимости логировать и продолжать (например, вкладка Servers подхватит при следующем открытии или при старте sing-box).

## Критерии приёмки

- После нажатия Update и успешного обновления конфига вызывается `ReloadClashAPIConfig()`.
- Вкладка Servers и селекторы после Update отражают данные из обновлённого `config.json` без рестарта sing-box/лаунчера (в рамках того, что даёт перечитывание Clash API из файла).
- Существующее поведение при ошибке Update не меняется.
- Рестарт sing-box после Update не требуется и не добавляется.

## Замечания

- sing-box сам по себе не перезагружает полный конфиг по Clash API (нет такого endpoint); меняется только состояние лаунчера (наш APIService), чтобы UI и выбор селекторов соответствовали новому config.json.
- `ReloadClashAPIConfig()` уже вызывается при старте sing-box (`process_service.startSingBox`); здесь добавляется вызов после успешного Update на главной вкладке.
